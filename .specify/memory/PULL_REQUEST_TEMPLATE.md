## PR Checklist
Please check if your PR fulfills the following requirements:

- [ ] The commit message follows our guidelines: https://github.com/tech4good-lab/causeway2/blob/develop/contributing-docs/commit-message.md
- [ ] If there is UI involved, the feature has been tested in multiple browsers and different window sizes
- [ ] Docs have been added / updated (for bug fixes / features)

#### Self-Review: 
Please be honest with your review as this is part of your grade.

- [ ] There are no typos in your code or comments
- [ ] Variable names are meaningful and clear
    - [ ] Variable names are in lower camel case (e.g. `userName`)
    - [ ] HTML/SCSS class names are in kebab case (e.g. `learn-container`)
- [ ] Ensure code, **especially any generated by AI**, aligns with the repository’s coding standards as detailed in our [contributing docs](https://github.com/tech4good-lab/causeway2/blob/develop/contributing-docs/coding-standards.md)
- [ ] Docstring formatting should be specific for functions, variables, inputs, tests, and outputs
- [ ] Checks specifically for functions:
    - [ ] Generally, add JSDoc tags (`@param {type} name - description`, `@returns`, etc.) where applicable
    - [ ] For simple functions, include when it triggers and what it does
    - [ ] For very complex components:
        - Add a comment at the top explaining the purpose of the code
        - Then also include inline comments to explain code (e.g. `// Set the focus and cursor on the new file`)
- [ ] Checks specifically for HTML:
    - [ ] For complex views, have comments explaining the structure (e.g. `<!-- stackblitz output -->`)
- [ ] Ensure all the `.ts` organizational comments (e.g. INPUTS AND OUTPUTS, EVENT HANDLING, OTHER, etc.) are present
- [ ] Remove unused items:
    - [ ] Imports on top of new files
    - [ ] Variables 
    - [ ] Functions
    - [ ] Old commented code
    - [ ] empty `ngOnInit()` or `constructor()` methods
- [ ] Ensure `ng lint` has been run and that all issues have been fixed (best practice is to run it before each commit; please confirm this was done before you request a PR review)
- [ ] Run `ng build` and confirm the project compiles without errors
- [ ] Reaffirm that the PR title and commit messages follow our [commit message guidelines](https://github.com/tech4good-lab/causeway2/blob/develop/contributing-docs/commit-message.md)
    - [ ] Include a scope for commit types that require it (e.g., `feat(main)`)
    - [ ] Add a commit description in present tense
- [ ] Include the issue number in the branch name and PR description
